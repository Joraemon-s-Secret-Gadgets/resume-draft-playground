# LLM Resume Draft Playground

## 1. 프로젝트 개요

이 프로젝트는 **사용자의 이력 정보와 유사한 배경을 가진 지원자들의 자소서 예시를 참고해, LLM이 자기소개서 초안을 생성하는 방식**을 실험하기 위한 저장소입니다.

현재 단계는 **사전조사 및 프로토타입 검증 단계**이며, 완성형 서비스 구현보다는 아래 내용을 우선적으로 확인하는 데 목적이 있습니다.

- 로컬 환경에서 오픈 LLM을 활용해 초안 생성이 가능한지
- 예시 자소서를 어떤 방식으로 넣어야 더 나은 초안이 생성되는지
- 프롬프트 엔지니어링만으로도 자소서 품질을 얼마나 개선할 수 있는지
- 향후 유사 이력 기반 추천 구조로 확장할 수 있는지

---

## 2. 프로젝트 배경

기존의 자기소개서 생성 방식은 일반적인 문장 생성에 머무르는 경우가 많아,  
실제 지원자의 경험과 직무 문항에 맞는 **설득력 있는 초안**을 만드는 데 한계가 있습니다.

본 프로젝트는 다음과 같은 아이디어에서 출발합니다.

1. 사용자로부터 이력과 지원 문항을 입력받는다.
2. 내부 데이터에서 사용자와 **유사한 이력**을 가진 사례를 찾는다.
3. 그중 **점수가 높았던 자소서**를 참고 사례로 활용한다.
4. LLM이 참고 사례의 **구조와 강조 포인트**를 반영하여 새로운 초안을 생성한다.
5. 이후 사용자 피드백과 추가 개선 로직을 통해 초안을 다듬는다.

현재 저장소에서는 이 전체 구조 중,  
우선 **예시 자소서를 입력으로 넣고 초안을 생성하는 최소 실험 단위**를 구현하고 있습니다.

---

## 3. 현재 단계

현재는 **사전조사 단계**입니다.

즉, 아래와 같은 항목을 중심으로 탐색하고 있습니다.

- 어떤 오픈 LLM을 사용할지
- API 비용 없이 로컬에서 실험 가능한지
- VSCode + Ollama 환경에서 구현이 가능한지
- 예시 자소서 기반 생성이 실제로 의미 있는 결과를 내는지
- 프롬프트 설계만으로 결과 품질을 어느 정도 통제할 수 있는지

아직 다음 항목들은 본격적으로 구현되지 않았습니다.

- 실제 유사 이력 검색 로직
- 고득점 자소서 자동 선별 로직
- 데이터베이스 연동
- 평가 점수 기반 추천 시스템
- 파인튜닝/LoRA 적용
- 사용자 인터페이스

---

## 4. 현재 실험 목표

현재 프로토타입의 목표는 다음과 같습니다.

- **입력**
  - 지원자 배경
  - 지원 직무
  - 자소서 문항
  - 작성 조건
  - 참고용 예시 자소서

- **처리**
  - 예시 자소서의 공통 장점과 전개 구조를 분석
  - 분석 결과를 바탕으로 자기소개서 초안 생성
  - 설명문이 아니라 자소서 문항에 대한 답변 형태로 출력되도록 통제

- **출력**
  - 자기소개서 초안

즉, 지금은 **“유사 우수 자소서 기반 초안 생성”의 가능성을 확인하는 단계**라고 볼 수 있습니다.

## 6. 모델 선택 배경

초기 실험 단계에서는 비용 문제를 고려해 **상용 API 대신 로컬에서 실행 가능한 오픈 LLM**을 사용하는 방향으로 정리했습니다.

그중 현재는 `qwen2.5:7b`를 사용하고 있습니다.

선택 이유는 다음과 같습니다.

- 로컬 환경에서 실행 가능
- API 비용이 들지 않음
- 비교적 긴 입력을 받을 수 있음
- 한국어 포함 다국어 처리 성능이 무난함
- 프롬프트 기반 구조 통제가 어느 정도 가능함

다만 현재까지 확인한 바로는,  
모델이 아무 제약 없이 생성할 경우 **자기소개서 초안보다는 설명문이나 경험 요약문 형태로 출력되는 경향**이 있어,  
프롬프트 엔지니어링이 매우 중요하다고 판단했습니다.

## 7. 현재 구현 방식

현재 코드는 **2단계 생성 구조**를 기준으로 실험하고 있습니다.

### 1단계: 예시 자소서 분석
입력된 예시 자소서들을 바탕으로 다음 요소를 추출합니다.

- 공통 장점
- 전개 구조
- 강조 포인트
- 피해야 할 표현

### 2단계: 초안 생성
1단계 분석 결과와 지원자 배경, 직무, 문항 정보를 함께 사용해 자기소개서 초안을 생성합니다.

이 구조를 사용하는 이유는,  
예시 자소서 원문을 그대로 생성 프롬프트에 넣는 경우 모델이 문장을 직접 따라가거나 단순 요약문을 만들 가능성이 높기 때문입니다.

따라서 지금은 **예시 자소서를 그대로 모방하는 방식이 아니라, 예시의 패턴만 추출해 적용하는 방식**을 우선 실험하고 있습니다.

## 8. 현재까지 확인한 문제점

사전 실험을 진행하면서 아래와 같은 문제를 확인했습니다.

### 1. 설명문 형태로 출력되는 문제
모델이 자기소개서가 아니라 프로젝트 경험을 정리한 설명문처럼 작성하는 경향이 있었습니다.

### 2. 첫 문장에서 강점이 분명히 드러나지 않는 문제
문항이 강점을 묻고 있음에도,  
생성 결과가 배경 소개나 프로젝트 개요로 시작하는 경우가 많았습니다.

### 3. 행동보다 해석 중심으로 서술되는 문제
지원자가 실제로 무엇을 했는지보다  
무엇을 느꼈는지, 무엇을 깨달았는지 위주로 쓰이는 경우가 많았습니다.

### 4. 예시 자소서와 사용자 배경이 지나치게 유사한 문제
테스트용 더미 데이터를 구성하는 과정에서  
`example_essays`와 `applicant_background`의 내용이 지나치게 비슷해져,  
유사 사례 참고라기보다 동일한 사례를 재서술하는 것처럼 보이는 문제가 있었습니다.

### 5. 출력 형식이 흔들리는 문제
bullet, 요약, 설명형 문단 구조가 섞이면서  
최종 제출용 자소서 초안으로 보기 어려운 결과가 나오는 경우가 있었습니다.

## 9. 현재 개선 방향

위 문제를 해결하기 위해 현재 다음 방향을 중심으로 실험 중입니다.

- 예시 자소서를 더 다양한 유형으로 구성
- 프롬프트에서 첫 문장 구조를 강하게 제한
- 자소서 문체와 설명문 문체를 명확히 구분하도록 지시
- 문제 상황 / 본인의 행동 / 판단 / 배운 점 / 직무 연결이 모두 포함되도록 강제
- 금지 표현 및 금지 출력 형식 설정
- 출력 결과가 기준을 어기면 재생성하는 방식 검토

## 10. 향후 계획

사전조사 이후에는 아래 방향으로 확장할 예정입니다.

### 1. 입력 구조 개선
- 하드코딩된 더미 데이터 대신 JSON 또는 파일 입력 방식으로 전환

- 내부 데이터와의 연결 방식 구체화

### 3. 생성 품질 개선
- 프롬프트 엔지니어링 고도화
- 출력 검사 및 재생성 로직 추가
- 모델 비교 실험

### 4. 모델 고도화
- 필요 시 LoRA / 파인튜닝 가능성 검토
- 한국어 자소서 생성 품질에 적합한 모델 비교

## 11. 현재 저장소의 의미

이 저장소는 완성된 서비스 구현체가 아니라,  
**“유사 이력 기반 자기소개서 초안 생성 시스템”의 가능성을 검증하기 위한 실험 저장소**입니다.

즉, 현재는 다음 질문에 답하기 위한 단계입니다.

- 오픈 LLM만으로도 초안 생성이 가능한가?
- 예시 자소서를 어떻게 넣어야 결과가 좋아지는가?
- 단순 생성이 아니라, 유사 사례 기반 구조를 만들 수 있는가?
- 프롬프트 엔지니어링만으로 어느 정도까지 품질을 끌어올릴 수 있는가?
